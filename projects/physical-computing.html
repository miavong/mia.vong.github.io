Technical Implementation

Using many booleans and different functions to map the game's status, I built a prototype and wrote the code to execute this game.

// Arduino C++ — game state logic

// Red LED on: game stopped, can present (lightThreshold <150)
bool gameRunning = false;
bool gameFinished = false;

// Can lifted (lightThreshold >150) → countdown + green LED
canOn = false;
countdownSequence();
startGame(); // gameRunning = true

// Can returned → timer stops, Red LED on
canOn = true;
gameRunning = false;
gameFinished = true;
stopGame();

// 7-seg shows elapsed time for 5s, then end-sequence message
displayElapsedTime();
finishSequence(); // based on elapsed time

// Game resets to 0.0

04Solution

The power of a photoresistor

Problem

I needed to track whether a can/beverage was present in the starting hold or not to start/stop/track the user's performance. Variables like weight or distance would have to be calibrated before each use, halting the flow of the game.

Solution

Thinking outside of the box... but really more like inside of the box — I used a photoresistor (a passive semiconductor component whose resistance decreases as light intensity increases) to measure whether or not the beverage was present. Using a LED light and a photoresistor inside the box, I created a controlled environment where the beverage acted as a gate to trigger the timer.

06Reflection

What I learned

This project taught me that creative constraints breed creative solutions. Instead of reaching for an expensive sensor, stepping back and thinking about the physics of the problem led me to a simple, reliable answer — a photoresistor inside a cardboard box. Sometimes the best engineering is the kind that makes you laugh at how obvious it was in hindsight.
